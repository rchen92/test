sysdba sysoper --->dba
sysdba ---> sys
sysoper ---> sys，system
dba用户：就是具备sysdba和sysoper权限的用户
dba用户的身份认证：
	1、操作系统身份认证（本地认证）
		sqlplus / as sysdba
	2、密码文件身份认证
		sqlplus sys/oracle@orcl as sysdba

密码文件位置
[oracle@oracle ~]$ ls $ORACLE_HOME/dbs/orapw$ORACLE_SID
/u01/app/oracle/product/11.1.0/dbhome_1/dbs/orapworcl

密码文件认证还是OS认证，取决于remote_login_passwordfile=none|exclusive|shared
	none--不使用密码文件认证
	execlusive--要使用密码文件认证，自己独占使用
	shared--要密码文件认证，不同实例dba用户可以共享密码文件
	默认是execlusive
和$ORACLE_HOME/network/admin/sqlnet.ora
	SQLNET.AUTHENTICATION_SERVICES=none|all
		none-->关闭OS认证，只能密码文件认证
		all-->关闭本机密码文件认证，采用OS认证，但远程（异地/异机）可以

密码文件建立
[oracle@oracle ~]$ orapwd
Usage: orapwd file=<fname> entries=<users> force=<y/n> ignorecase=<y/n> nosysdba=<y/n>

  where
    file - name of password file (required),	--->密码文件的名字 orapw<sid>
    password - password for SYS will be prompted if not specified at command line,		--->sys用户的密码
    entries - maximum number of distinct DBA (optional),		--->可以有多少的sysdba，sysoper权限用户放到密码文件中去
    force - whether to overwrite existing file (optional),
		--->默认值为n，表示如果本来有密码文件，则创建一个新的密码文件会失败，显示密码文件已存在
		orapwd file=orapworcl password=uplooking force=y
    ignorecase - passwords are case-insensitive (optional),		--->
    nosysdba - whether to shut out the SYSDBA logon (optional Database Vault only).		--->
    
  There must be no spaces around the equal-to (=) character.

密码文件的修改
	alter user sys identified by <>;
	grant sysdba to <>;
	grant sysoper to <>;
	revoke sysdba|sysoper from <>;


	SQL> grant sysdba to scott;
	Grant succeeded.
	SQL> create user s1 identified by s1;
	User created.
	SQL> grant sysoper to s1;
	Grant succeeded.
	SQL> select * from v$pwfile_users;
