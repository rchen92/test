在有冷备份的情况下
1、丢失数据文件
	a.丢失system表空间，则先全备数据库，然后再使用最近的冷备还原到目的路径，然后启动数据库，完成恢复到冷备时刻，冷备后变化的数据只有在归档模式下才能恢复到损坏的时刻。由于是非归档（所以才使用的冷备），只能恢复到最近冷备份时刻
	b.sysaux丢失，数据库启动，将sysaux数据文件脱机，导出数据，然后在新建数据库，导回数据。
						  或者用冷备数据还原，加隐参启动数据库忽略一致性验证
	c.undo丢失，删除旧的，建立新的
	d.自己建立的表空间，脱机该数据文件，然后启动数据库，导出其他可用的数据
	e.temp丢失，建立新的
2、日志文件丢失
	通过resetlogs恢复（重建）日志文件
3、控制文件丢失
	建立新的控制文件
4、参数文件丢失
	使用冷备恢复即可
5、密码文件丢失
	使用备份恢复或者建立新的密码文件


--冷备下非归档模式，控制文件全部丢失，恢复方法：
--自冷备以后没有发生数据文件增加减少的操作，则使用老的控制文件还原回去，并启动到mount得到建立控制文件的语句，关闭数据库shutdown abort，然后删除被还原的控制文件。修改建立控制文件的语句，执行语句，得到控制文件，并恢复数据库到损坏之前的时刻，这样数据不会丢失。执行全库备份
SQL> alter database backup controlfile to trace as '/backup/ctl.sql';
Database altered.
SQL> ho rm /u01/app/oracle/oradata/orcl/control*
SQL> shutdown abort
ORACLE instance shut down.
SQL> exit
